# 特许经营法律风险情报监控 - 优化版

这个版本不是普通“新闻关键词提醒”，而是更偏向律师使用的“特许经营法律风险线索监控”。

## 这版改了什么

1. 改成打分筛选：不再是看到“连锁”两个字就推送，而是按“特许经营、行政处罚、加盟合同、退费、未备案、虚假宣传、重点品牌”等因素综合打分。
2. 加入律师视角：推送里会显示风险类型、命中词、命中品牌和律师关注角度。
3. 修复历史记录逻辑：只有真正准备推送的内容才会写入历史，避免“没推送但以后也不推”的问题。
4. 保护企业微信机器人地址：机器人地址放在 `config.json` 或环境变量里，不再写死在代码里。
5. 加入品牌词库：已经放入一批重点连锁经营品牌，并接入 CCFA TOP300 品牌词库。
6. 增加正文补抓：标题或摘要疑似相关时，会尝试打开原文补抓正文，提高发现“处罚书正文命中特许经营”的概率。
7. 增加每日汇总：可每天固定时间推送一份更像情报简报的内容。
8. 增加时效过滤：默认只推送最近 30 天内能识别发布时间的内容，识别不到发布时间的内容默认不推送。
9. 增加重复过滤：除了链接去重，也会对标题去重，减少同一新闻换来源反复推送。

## 第一次使用

### 1. 安装依赖

在当前文件夹运行：

```bash
pip install -r requirements.txt
```

如果你是在 Trae 环境里运行，可能需要：

```bash
pip install -r requirements.txt --break-system-packages
```

### 2. 配置企业微信机器人

打开 `config.json`，找到：

```json
"wecom_webhook": ""
```

把空字符串改成你的企业微信机器人地址：

```json
"wecom_webhook": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key"
```

也可以不填。不填时程序会运行，但只在日志里预览，不会真正推送到群里。

### 3. 运行

```bash
python3 franchise_monitor.py
```

程序启动后会立刻执行一次抓取，然后每隔 `config.json` 里的 `fetch_interval_minutes` 分钟执行一次。

只运行一轮，不常驻：

```bash
python3 franchise_monitor.py --once
```

干跑测试，不推送企微，也不写入历史：

```bash
python3 franchise_monitor.py --once --dry-run
```

只推送每日汇总：

```bash
python3 franchise_monitor.py --daily-summary
```

## 为什么企微机器人地址不要写死在代码里

企业微信机器人地址相当于“可以往你的群里发消息的钥匙”。

如果写死在代码里，风险是：

1. 你把项目发给别人时，对方能看到这个地址。
2. 你不小心上传到 GitHub 时，别人可能拿它往你的群里发垃圾信息。
3. 发给 AI、同事或外包人员时，容易无意泄露。
4. 后续如果机器人地址失效或更换，需要改代码，不方便维护。

所以更推荐放在 `config.json` 或本机环境变量里。

## 如何加入 CCFA TOP300 名录

你后续把 CCFA TOP300 名单发给我，我可以帮你直接导入。

如果你想自己加，可以打开 `config.json`，找到：

```json
"ccfa_top300_brands": []
```

改成：

```json
"ccfa_top300_brands": [
  "品牌A",
  "品牌B",
  "品牌C"
]
```

注意每个品牌名要用英文双引号包起来，中间用英文逗号分隔。

## 推荐关键词逻辑

这版会重点关注：

```text
商业特许经营、特许经营、特许人、被特许人、加盟合同、加盟商、加盟费、行政处罚、处罚决定书、责令改正、罚款、虚假宣传、未备案、退费、解除合同、判决、维权、投诉
```

其中“特许经营”“行政处罚”“处罚决定书”“未备案”“加盟合同”等词权重更高。

## 推送内容怎么看

推送里会出现：

```text
[高优先级] 标题
风险类型：行政处罚 / 加盟合同纠纷 / 招商宣传风险
命中品牌：喆啡
命中词：特许经营、行政处罚、罚款
律师关注：核查处罚依据、违法事实、处罚机关和是否可类比到客户招商合规。
```

你可以把它当作“选题雷达”，不是最终法律意见。真正对外输出前，仍建议点开原文、处罚决定书或判决书复核。
